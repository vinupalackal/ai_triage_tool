# Agent Grounded Context (Hallucination Guardrails)

Last reviewed: 2026-07-09

## Purpose
This document is a hard boundary for AI responses about this repository. It separates currently working behavior from planned design so responses do not over-claim.

## Current Reality (Implemented)
- Streamlit app with six tabs is implemented in app.py.
- Ingestion is implemented for:
  - Source code from local folder and git URL.
  - Logs from local folder and Jira issue attachments.
  - Documents from file upload and URL with extracted text sidecar files.
- Tracking is implemented in DuckDB via utils/storage.py:
  - artifacts table and listing/count APIs.
  - index_state table helpers for incremental indexing bookkeeping.
  - component tagging and component-based artifact lookup.
  - component-aware cache checks used by Source code and Documents tabs.
- Tests exist for app behavior, ingestion modules, and storage layer.

## Designed But Not Implemented (Do Not Present As Working)
- Agent execution loop is not implemented.
  - agent/loop.py investigate() raises NotImplementedError.
  - agent/tools.py dispatch() raises NotImplementedError.
- Indexing layer is not implemented.
  - indexing/log_templating.py raises NotImplementedError.
  - indexing/code_chunking.py raises NotImplementedError.
  - indexing/doc_embedding.py raises NotImplementedError.
- Retrieval layer is not implemented.
  - retrieval/hybrid_search.py raises NotImplementedError.
  - retrieval/graph_query.py validates supported query shapes, then raises NotImplementedError.

## Documentation vs Code
- The architecture/design/requirements documents describe the MVP target state (including indexing, retrieval, and full AI investigation flow).
- The codebase currently implements ingestion plus tracking, not the full target MVP behavior.
- Therefore, responses must always label capabilities as one of:
  - implemented now
  - designed/planned
  - deferred/post-MVP

## Safe Claims You Can Make
- The app can ingest and track artifacts from local and remote sources.
- Component tagging and component-based cache skipping are operational.
- The Investigate tab is wired to the real interface but intentionally reports not-yet-built status.

## Claims You Must Avoid
- Do not claim operational vector search, BM25, graph traversal, symbolication, blame-history lookup, or multi-step agent reasoning.
- Do not claim root-cause answers are generated by the current app.
- Do not describe AI-FR capabilities as completed unless code is implemented and validated.

## Response Template for Accuracy
When asked what the system can do, use this structure:
1. Implemented now: list only verified runtime behavior.
2. Designed next: list stubbed modules and intended behavior.
3. Gaps and risk: explain what evidence is currently unavailable because indexing/retrieval/agent are stubbed.
4. Next concrete step: point to the first unimplemented module required to unlock the requested capability.
